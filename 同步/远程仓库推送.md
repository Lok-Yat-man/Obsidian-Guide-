在 **Obsidian Git** 中，插件原生支持的是 **按间隔自动 commit-and-sync**，也就是“每隔 X 分钟执行一次提交、拉取、推送”；它**不是按固定钟点**（例如每天 3:00）触发。定时触发需要 Windows：任务计划程序，或者Linux：`cron`。

### 第一步：环境准备

1. **创建 GitHub 私有仓库：** 在 GitHub 上创建一个名为 `MyNotes`（或任何你喜欢的名字）的仓库，务必设为 **Private**。
    
2. **配置 SSH 免密登录：** 确保你的工位电脑已经生成并配置了 GitHub 的 SSH Key，以便脚本在后台运行时无需手动输入密码。[[配置SSH密钥]]
    
3. **本地初始化：** 在工位电脑的 Obsidian 库根目录下执行：
    
    ```
    git init
    git remote add origin git@github.com:你的用户名/仓库名.git
    ```
    

---

### 第二步：配置 `.gitignore` 和 `.stignore`（至关重要）

为了防止 Syncthing 的临时文件或 Obsidian 的冗余缓存被推送到 GitHub，请在库根目录创建 `.gitignore` 文件，并写入以下内容：


```
# 忽略 Syncthing 冲突文件
*.sync-conflict-*
.stversions/

# 忽略 Obsidian 缓存与工作区状态（避免频繁提交无意义的 JSON 变动）
.obsidian/workspace.json
.obsidian/workspace-mobile.json
.obsidian/cache/
.obsidian/plugins/obsidian-git/ # 如果你用了 Git 插件，忽略其自带日志

# 忽略系统文件
.DS_Store
Thumbs.db
node_modules/
```

如果 Git 插件正在执行 `commit`，而 Syncthing 恰好在同步 `.git` 文件夹里的索引，可能会报错。**建议：** 在 `.stignore`（Syncthing 的忽略文件）中增加一行 `.git/`。
既然打算只在“中心节点”电脑上运行 Git 备份，那么其他设备（手机、笔记本）其实没必要同步 `.git` 文件夹。这样可以显著提高手机端的同步速度，并避免 Git 索引损坏。

---

### 第三步：编写自动化脚本

根据你工位电脑的操作系统，选择对应的脚本方案。

#### 方案 A：Windows (PowerShell)

在库文件夹外创建一个 `backup_notes.ps1` 文件：

PowerShell

```
# 进入笔记目录
cd "D:\Your\Obsidian\Vault"

# 执行 Git 操作
git add .
$currentDate = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
git commit -m "Auto-backup: $currentDate"
git push origin main
```

#### 方案 B：Linux (Shell Script)

创建 `backup_notes.sh`：

Bash

```
#!/bin/bash
cd /path/to/your/obsidian/vault
/usr/bin/git add .
/usr/bin/git commit -m "Auto-backup: $(date '+%Y-%m-%d %H:%M:%S')"
/usr/bin/git push origin main
```

### 第四步：设置定时任务

#### 如果是 Windows：使用任务计划程序 (Task Scheduler)

1. 搜索并打开“任务计划程序”。
    
2. **创建任务：** 名称设为 `Obsidian_Auto_Backup`。
    
3. **触发器：** 新建，选择“每天”，时间设在凌晨（或你下班后的时间）。
    
4. **操作：** 新建，程序或脚本填写 `powershell.exe`，参数填写 `-ExecutionPolicy Bypass -File "C:\路径\to\backup_notes.ps1"`。
    
5. **条件：** 勾选“只有在网络连接可用时才启动”。
    

#### 如果是 Linux：使用 Crontab

输入 `crontab -e`，添加一行（例如每天凌晨 3 点执行）：

Bash

```
0 3 * * * /bin/bash /path/to/your/backup_notes.sh
```

### 总结与风险规避

- **冲突优先：** 所有的同步冲突依然由 Syncthing 在本地解决。Git 只负责记录最终的结果。
    
- **二进制文件：** 如果你的笔记库里有大量 PDF 或大图，GitHub 仓库可能会迅速膨胀。建议对此类文件保持克制，或者利用 Git LFS。
    
- **多端拉取：** 既然已经在工位电脑实现了 `Push`，如果你在笔记本上想回溯版本，只需执行 `git pull` 即可。
